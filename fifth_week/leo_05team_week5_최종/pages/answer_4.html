[해설] 미디어 쿼리를 분기하는 스크린 사이즈

Tailwind CSS는 아래와 같은 분기(브레이크 포인트)를 기본으로 제공합니다.
실제 메인페이지의 CSS코드에서 '@media (min-width:'로 검색하면 5가지의 모든 사이즈를 확인할 수 있었으나,
purgeCSS 예외처리의 대상이 아니라서
(/*! purgecss start ignore */ ~ /*! purgecss end ignore */ 사이에 없다는 의미)
페이지에서 사용하는 사이즈만 호출할 것으로 보입니다.

출처: https://tailwindcss.com/docs/screens
  • 'sm' ➡️ @media (min-width: 640px) { ... }
  • 'md' ➡️ @media (min-width: 768px) { ... }
  • 'lg' ➡️ @media (min-width: 1024px) { ... }
  • 'xl' ➡️ @media (min-width: 1280px) { ... }
  • '2xl' ➡️ @media (min-width: 1536px) { ... }
다만 Tailwind CSS 예제 페이지에서는 'lg:max-w-5xl'로 제한하였기 때문에, 1024px 이상으로 폭을 늘일수 없게 막아두었습니다. 그래서 sm, md, lg 분기 만을 사용하였다고 판단했고 정답은 아래와 같습니다.
  • 'sm' ➡️ @media (min-width: 640px) { ... }
  • 'md' ➡️ @media (min-width: 768px) { ... }
  • 'lg' ➡️ @media (min-width: 1024px) { ... }
참조: max-w-5xl ➡️ max-width: 64rem; /* 1024px */
